  • Technical Projects

    • Art Reference Catalog Fullstack App

      2022

      After drawing and painting for several years, I accumulated hundreds of reference images on my local harddrive. There were countless times in which I would want to find a specific image to reference, but I would then have to look through hundreds of images to find it. Even with all the images categorized, this was often very difficult. So I decided to build a full stack app to save the image names along with user added tags. The user can upload an image and enter information for the image: character name, source name, body type, NSFW filter, and tags. You can search for images via this information or view random images with or without a filter. You can also screen for NSFW images if that is something you want to avoid. The app is fullstack with an adaptive front-end and uses cloud storage via MongoDB atlas.

    • Custom Constructed Language Dictionary

      2020

      In 2019, I got interested in world-building and writing stories for these fictional worlds. Part of that was developing written and spoken languages. One difficulty in that, is remembering definitions of the potentially hundreds of words your conlang may have. Initially, I was storing everything on a google doc, but this quickly became too complicated. So instead, I built a CLI tool that allows you to add, edit, and delete words and their definitions. There was no need for a traditional DB solution, so it is simply a collection of specific JSON operations. However, it was really helpful and is an excellent example of successfully filling a need and creating a time-saver with what I have learned.

    • RetinaNet Object Detection on MBARI Station M Images

      2019

      This was my senior year capstone project in which we trained a deep learning model on the Google Cloud Platform in order to help deep sea researchers identify and track benthic life. The Monterey Bay Aquarium Research Institute had seafloor imaging stations that would capture an image of the sea floor every hour for an entire year. Afterwards, they would have ~8000 images to look through to identify any lifeforms captured by the camera. This is a very time-consuming task and very tedious. Our project was to provide an automated solution using Deep learning and Computer Vision. We were to train a, at the time, bleeding edge DL model called RetinaNet. The model was so new, much of the cloud infrastructure on GCP was not equipped to run this model and we were forced to build much of those processes. I specifically worked on the tensorflow record transfer and training/inference portion of the pipeline.

    • Audio Reactive PBR Render Engine

      2018

      I wanted to learn how to make a game engine with my friend. So we set out to build one using OpenGL and GLSL. Following a tutorial, we managed to build a very simple engine that could load objects, apply shaders, and allow camera movement. Afterwards, I wanted to do something fun and decided to make an audio visualizer using shaders as the frequency response. Built in the same stack, you can pass an audio file to the program, it will then play the file and use FFT to extract frequency information. It then bins the frequency range and each bin is attached to some parameter in the lighting shaders. I didn't know too much about how to make 3D lighting look good at the time and the frequency response is not that good, but I was still able to get a result.
