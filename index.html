  • Technical Projects

    • Personal Spotify Wrapped

      2024/25

      Inspired by a Tiktok and a want for a more visible portfolio, this project uses Spotify’s web API to create a personal Spotify Wrapped written entirely in Python. I do plan to revisit near the end of 2025 to see what has changed and to compare results with the official Spotify wrapped. This notebook looks at my listening history and generates an image with visuals that change in accordance with the data. It generates an “album map” which is the top 4 listened albums averaged together and then passed through a median filter to create an interesting effect. We also use the average color of this album map to decide what color theme to use for the final image. Lastly we use a histogram of the songs’ popularity field to make the background a little more interesting. If you listen to less popular music, the bars will be right skewed and more left skewed if you tend to favor the top of the billboards. As of writing (1/6/25), there is only about two weeks worth of data. Data is collected via a script hosted on a remote Ubuntu server that pulls my most recently played songs from Spotify web API ever hour. The script saves the most recently played songs to a csv file using the played_at field to make sure there is no overlapping or duplicated data and to only append unrecorded data while also doing some initial cleaning. We then simply pull the csv file that now contains my entire listening history for us in this notebook. Initially, I planned on doing an in depth statistical analysis for my listening habits using the number of times I listened to a song as the response variable. However a few days before I started working on this project, Spotify deprecated their audio feature API endpoint. This significantly decreased the amount of numeric features that could be used in a statistical model without having to search for, download, then process thousands of songs myself. We could derive more features from the text data. However, I wanted to concentrate on the application of linear models rather than NLMs. My initial concern was that, within black box of neural models, it may simply favor similary to or matches with the most popular artists and song titles. I would say that a band having a name similar to my most listened to band is not grounds for wether I would listen to a song a lot. For instance, let’s consider the two bands “Ghost” and “As Ghosts”. Their names are very similar and probably have high cosin similarity, but I have not listened to a “Ghost” song in months whereas “As Ghosts” has become one of my favorite artists in the past month. In leue of working statistical models, I decided to instead challenge myself to make something that I would typically use Photoshop for except entirely in Python.

    • Art Reference Catalog Fullstack App

      2022

      After drawing and painting for several years, I accumulated hundreds of reference images on my local harddrive. There were countless times in which I would want to find a specific image to reference, but I would then have to look through hundreds of images to find it. Even with all the images categorized, this was often very difficult. So I decided to build a full stack app to save the image names along with user added tags. The user can upload an image and enter information for the image: character name, source name, body type, NSFW filter, and tags. You can search for images via this information or view random images with or without a filter. You can also screen for NSFW images if that is something you want to avoid. The app is fullstack with an adaptive front-end and uses cloud storage via MongoDB atlas.

    • RetinaNet Object Detection on MBARI Station M Images

      2019

      This was my senior year capstone project in which we trained a deep learning model on the Google Cloud Platform in order to help deep sea researchers identify and track benthic life. The Monterey Bay Aquarium Research Institute had seafloor imaging stations that would capture an image of the sea floor every hour for an entire year. Afterwards, they would have ~8000 images to look through to identify any lifeforms captured by the camera. This is a very time-consuming task and very tedious. Our project was to provide an automated solution using Deep learning and Computer Vision. We were to train a, at the time, bleeding edge DL model called RetinaNet. The model was so new, much of the cloud infrastructure on GCP was not equipped to run this model and we were forced to build much of those processes. I specifically worked on the tensorflow record transfer and training/inference portion of the pipeline.

      2018

      I wanted to learn how to make a game engine with my friend. So we set out to build one using OpenGL and GLSL. It then bins the frequency range and each bin is attached to some parameter in the lighting shaders.     • 2D Texture Eye Rig in Blender

      2025

      In the past few months I’ve gotten really into 3D animation. I’ve dabbled in 3D modeling and Blender before but have started spending a lot of time in the program. When I first got into 3D, it was for the purpose of making 2D animation. The idea was to create a 3D to 2D pipeline where 3D models would do the heavy lifting for animation and I would draw over each frame in 2D to try and preserve authentic 2D stylings. However, this proved too time consuming and my 3D modeling skills are still not where I want them to be. Instead, I’ve opted to use stylized low-poly models that use 2D textures for the face. An issue I found immediately was the need to draw each eye shape for every direction a character looks. My solution was to create a 2D Texture eye rig. A group of numbered textures form a grid of pupils that cover every direction a character can look. We then create a driver that looks at the X,Y coordinates of a bone and map them to its corresponding pupil texture. A texture with transparency to allow the pupil to always be visible is then placed on top of the selected pupil texture. This allows the animator to independently control the eye’s expression and eye direction without the need to draw an excessive amount of eye textures.
